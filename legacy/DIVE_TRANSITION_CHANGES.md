# 潜る演出のリニューアル (dive-flow-19)

「潜る」演出を **Canvas（実際の海が描かれている場所）の中で起こす** 方式。
対象は `legacy/` の静的プロトタイプ（公開先: `/Luminous-Tide/legacy/`）。

## これまでの経緯

- 16/17: 水中ビューのDOM要素を動かして潜航を表現 → 「ポップアップがバラバラ／カクカク」。
- 18: 演出をCanvasへ移しカクつきは解消。ただし線へズームして暗転する作りで「沈んでる感じがしない／水平線が消える」。
- **19（今回）**: 線を消さず、**明るいまま画面上方へ昇らせて水の断面（天井）として残す**。ズームをやめて**シーン全体をゆっくり上へ送る＝視点が下降**する動きに。速度も遅く。

## 演出（Canvas主導）

1. **集約**: 「潜る」で `diveProgress`(0→1) が一定速度で立ち上がる。夜光虫（粒子）が水平線へ吸い寄せられ、波の振幅が0に近づき、発光ラインが一本に集約・増光。
2. **下降**: シーン全体を上方向へ平行移動（`translate`）し、水平線を画面上部（`DIVE_LINE_TOP=44px`）へ昇らせる。**線は消さずキープ＝水の断面**。線より下は徐々に暗くなり「深く沈む」感を出す。
3. **Dive Log**: 線を越えたあと（`diveProgress>0.82`）に、水中画面を1枚で静かにフェードイン。Dive Log側の水面ライン（上端42px）と着地位置を合わせ、線が連続する。

## 今回の変更点（18→19）

### `legacy/app.js`
- 補間を**時間ベースの一定速度**へ: `DIVE_IN_RATE = 1/2.4`（潜降 約2.4秒）/ `DIVE_OUT_RATE = 1/1.5`（浮上 約1.5秒）。以前の指数補間（速すぎた）を廃止。
- `frame()`: ズーム＋全画面暗転をやめ、**`translate` でシーンを上へ送る**だけに。水平線の着地位置 `DIVE_LINE_TOP=44`。
- 線より下だけを暗くする**deepening グラデーション**を追加（線は明るいまま残す＝水の断面）。
- 潜降中はシーンが上に動くため、背景塗りが下端を覆わなくなる → **画面クリアを追加**してゴースト（残像）防止。
- `updateDiveStage()` を新設し、Dive Log の表示・状態クリアを **`diveProgress` のしきい値で同期**（`setTimeout` を全廃）。
- `openDiveView()`/`closeDiveView()` は `diveTarget` を 1/0 にするだけのシンプル構成に。

### `legacy/index.html`
- キャッシュバスターを `dive-flow-18` → `dive-flow-19`。

## 微調整のツマミ

- 潜降の速さ → `DIVE_IN_RATE`（`1/2.4` の 2.4 を増やすと遅く）
- 線が昇りきる位置（水の断面の高さ）→ `DIVE_LINE_TOP`
- 線への吸い寄せの強さ → `updateParticles()` の `pull * 0.16`
- 線の明るさ → `drawWaterBase()` の `diveBoost = 1 + diveProgress * 3.4`
- 深みの暗さ・出方 → `frame()` の deepening グラデーション（`0.55 * deep` / `0.98 * deep`）
- Dive Log が出る間合い → `updateDiveStage()` の `diveProgress > 0.82`
